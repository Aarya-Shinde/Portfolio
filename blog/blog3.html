My Tech Journey: Learning Git & GitHub Without Losing My Mind

Posted on July 1, 2025 by Aarya Shinde

Introduction

If I had a rupee for every time someone told me “Use Git!”, I could probably fund my own startup by now. But I’ll admit it — for a long time, I avoided it. Git felt like one of those mysterious developer tools you *pretend* to know about until someone asks you to explain git rebase. Eventually, I decided enough was enough — time to actually learn it.

Why Git & GitHub?

In simple terms, Git is a version control system — a fancy way of saying “it remembers all your code changes so you can go back in time when you mess up.” GitHub is like Git’s social media — a place to store, share, and collaborate on code with others.

Before Git, I had folders named project_final, project_final_v2, and project_final_FINAL_REALLY_THIS_ONE. Now, Git keeps my history clean and my sanity intact.

First Steps

I started small — just creating a local repository and making a few commits. The first commands I learned became my everyday toolkit: 

# Initialize a Git repo
git init

# Check file status
git status

# Add changes
git add filename.txt

# Commit changes with a message
git commit -m "Initial commit"

Seeing those commits pile up felt strangely satisfying. Like I was giving my code a personal diary.

Pushing to GitHub

Once I got the basics, I linked my local repo to a GitHub repository. That’s when I started feeling like a “real developer.” Here’s what the magic looked like: 

# Connect local repo to GitHub
git remote add origin https://github.com/username/repo-name.git

# Push code to GitHub
git push -u origin main

Suddenly, my code was on the internet — versioned, safe, and ready to share.

Branching & Merging

Branches quickly became my favorite feature. They let me experiment with new features without breaking my main code. And if things went horribly wrong… I could just delete the branch and pretend it never happened. 

# Create and switch to a new branch
git checkout -b feature-branch

# Merge branch into main
git checkout main
git merge feature-branch

Common Mistakes & Lessons

  • Commit often: Small commits are easier to debug than one giant commit called “fixed stuff”.
  • Read error messages: Git’s errors are like riddles — cryptic but solvable.
  • Don’t fear the terminal: The CLI is your friend, not your enemy.
  • Pull before you push: Avoid conflicts by syncing with the remote repo first.

Resources I Used

  • Pro Git Book — Free and super detailed.
  • Learn Git Branching — Interactive and visual.
  • GitHub Docs — Everything you need straight from the source.

Conclusion

Learning Git & GitHub felt intimidating at first, but once I started, I realized it’s less about memorizing commands and more about understanding the workflow. Now, I can’t imagine coding without it. My advice? Just start small, mess around, and don’t be afraid to break things — Git will always have your back.
